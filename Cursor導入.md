## Cursor導入

### この章の位置づけ

Coworkは、Claudeに作業を依頼するには便利です。

一方で、フォルダやファイルを自分の手で直接開いて、文章を少し直したり、ファイル名を確認したり、Markdownを編集したりする用途には向いていません。

そのため、この章では `Claude業務フォルダ` を人間が直接編集するための道具として、Cursorを使います。

Cursorは本来、開発者向けのAIエディタです。ただし、Markdownやテキストファイルを編集するだけなら、非エンジニアでも十分に使えます。

この章では、プログラム開発ではなく、次のような使い方に絞ります。

- `Claude業務フォルダ` を開く
- Markdownファイルを手で編集する
- ファイルの場所や中身を確認する
- 必要に応じて、CursorのAIに文章の整理を手伝わせる

### 事前に確認すること

Cursorを入れる前に、次の点を確認してください。

| 確認項目 | 内容 |
| --- | --- |
| アカウント | Cursorの利用にはサインインが必要 |
| 対応OS | macOS、Windows、Linuxに対応 |
| Privacy Mode | 必要に応じて、設定からPrivacy Modeを有効にする |

Cursor公式のセキュリティページでは、Privacy ModeはFreeやProでも利用でき、有効にするとデータを学習に使わないと説明されています。

社外秘の資料、顧客情報、個人情報を扱う場合は、Cursorを使う前に会社のルールを確認してください。

### インストール手順

1. Cursorのダウンロードページを開く
    - https://cursor.com/download

2. 自分のOSに合うものを選ぶ
    - Mac
    - Windows
    - Linux

3. ダウンロードしたファイルを開き、画面の案内に従ってインストールする

4. Cursorを開き、アカウントでサインインする

5. Privacy Modeを有効にしておく

### Claude業務フォルダを開く

Cursorを開いたら、次の手順でフォルダを開きます。

1. Cursorを開く
2. `Home`を押す
3. `Open Folder`を選ぶ
4. `Claude業務フォルダ` を選ぶ
5. 右側にフォルダ一覧が表示されることを確認する
  - Cursorでは、右側のファイル名をクリックすると、中身を編集できます。

### ファイルを手で編集する

まずは、すでにあるMarkdownファイルを開いてみます。

1. 左側の `10_業務ルール` を開く
2. `営業メール作成ルール.md` を開く
3. `Markdown`を押す
4. 必要があれば、文章を少しだけ直す
5. 保存する

保存のショートカットは次の通りです。

| パソコン | 保存 |
| --- | --- |
| Mac | `Command + S` |
| Windows | `Ctrl + S` |

### 新しいファイルを作る

次に、手で新しいMarkdownファイルを作ります。

1. 左側の `10_業務ルール` を右クリックする
2. `New File`を選ぶ
3. ファイル名を入力する

```text
社内共有メモ作成ルール.md
```

4. 次の内容を入力する

```text
# 社内共有メモ作成ルール

## 基本ルール

- 結論を最初に書く
- 決定事項と未決事項を分ける
- 次にやることは担当者と期限を書く
- 推測で事実を書かない
```

5. 保存する

このように、Cursorを使うと、Coworkに頼まなくても自分の手でファイルを追加、編集できます。

### CursorのAIに手伝わせる

CursorにはAI機能があります。

文章の整理を手伝わせる場合は、次のように依頼します。

```text
開いている社内共有メモ作成ルール.mdを確認して、
ルールの順番を分かりやすく並べ替える案を出してください。

まだファイルは編集しないでください。
まず提案だけ表示してください。
```

慣れてきたら、編集まで依頼できます。

```text
社内共有メモ作成ルール.mdを編集して、
ルールを「書き出し」「本文」「次のアクション」の3つに分類してください。

編集後、変更点を簡単に説明してください。
```

### 差分を確認する

CursorでAIに編集させた場合は、変更内容を確認してから使います。

確認するポイントは次の通りです。

- どのファイルが変更されたか
- 消えてはいけない文章が消えていないか
- 顧客名、日付、金額などが勝手に変わっていないか
- 保存先が意図したフォルダになっているか
- 不明な情報を推測して書いていないか

Cursor公式のQuickstartでも、変更後にdiffを確認し、必要なチェックを実行する流れが紹介されています。

この手順書の範囲では、テストやビルドは不要です。Markdownファイルの中身を目で確認できれば十分です。
