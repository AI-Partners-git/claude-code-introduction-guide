## 6. スラッシュコマンドや細かい補足

### スラッシュコマンドとは

スラッシュコマンドとは、Claude Codeの作業画面で使えるショートカットです。

入力欄の先頭に `/` を入力すると、使えるコマンドの一覧が表示されます。

たとえば、設定を確認したいときは `/status`、先に進め方を確認したいときは `/plan` のように入力します。

```text
/status
```

普通の文章で依頼する代わりに、Claude Codeの操作をすばやく実行するためのものです。

### まず覚えるコマンド

最初からすべて覚える必要はありません。

まずは、次のコマンドだけ知っておけば十分です。

| コマンド | 使う場面 |
| --- | --- |
| `/status` | ログイン状態、モデル、接続状態などを確認したいとき |
| `/model` | 使用するAIモデルを確認・変更したいとき |
| `/usage` | 利用状況や上限を確認したいとき |
| `/plan` | いきなり作業させず、先に進め方を確認したいとき |
| `/compact` | 会話が長くなったので、内容を要約して続けたいとき |

迷ったら、個別のコマンド名を覚えるよりも、入力欄で `/` を押して、表示された一覧から選びます。

### よく使う流れ

実際の業務では、次のような流れで使います。

#### 状態や利用状況を確認する

最初に、Claude Codeがどのアカウント、モデル、接続状態で動いているかを確認したい場合は `/status` を使います。

```text
/status
```

利用状況や上限を確認したい場合は `/usage` を使います。

```text
/usage
```

使用するAIモデルを確認・変更したい場合は `/model` を使います。

```text
/model
```

モデルを変更すると、回答の速さや精度、利用量に影響する場合があります。

研修では、まず現在の状態を確認するところまでにして、モデル変更は必要になったときだけ行えば十分です。

#### 作業前に進め方を確認する

大きな修正をする前は、最初に `/plan` を使うと良いです。。

```text
/plan 営業資料を経営者向けに短く直す前に、修正方針を出してください。
```

このように依頼すると、Claude Codeはすぐに編集せず、先に進め方を整理します。

認識がズレていないか確認してから作業を任せることができます。

#### 会話が長くなったら整理する

同じ会話で何度もやり取りすると、過去の内容が増えていきます。

会話が長くなったときは、`/compact` を使います。

```text
/compact ここまでの決定事項と、次にやる作業だけ残してください。
```

これにより、必要な情報を残したまま会話を軽くできます。

### Skillもスラッシュコマンドから呼び出せる

Claude Codeで使えるSkillとして登録されている場合、Skillもスラッシュコマンドのように呼び出せます。

たとえば、`sales-email` というSkillを作っている場合、次のように入力します。

```text
/sales-email
```

または、通常の文章で次のように依頼しても構いません。

```text
「sales-email」を使って、営業メールを作ってください。
```

初心者の場合は、まず通常の文章で依頼し、慣れてきたら `/sales-email` のような呼び出し方を試すとわかりやすいです。

### 許可を求められたときの考え方

Claude Codeは、ファイルを編集したり、コマンドを実行したりするときに、許可を求めることがあります。

これは、意図しない変更を防ぐための確認です。

特に次の操作では、内容を確認してから許可します。

- ファイルの作成、編集、削除
- 外部サービスへの接続
- コマンドの実行
- GoogleカレンダーやGmailなど、コネクタで接続した外部サービスへの操作

よくわからない場合は、許可する前に次のように聞きます。

```text
この操作で何が変更されますか？
初心者にもわかるように説明してください。
```

説明を確認してから進めることで、不要な変更を避けられます。
