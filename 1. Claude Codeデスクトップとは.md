## 1. Claude Codeデスクトップとは

### チャット型AIとClaude Codeの違い

最近のチャット型AIでも、文章作成、要約、Web検索、資料のたたき台作成など、かなり多くのことができます。

では、Claude Codeは何が違うのでしょうか。

一番の違いは、Claudeを「会話する相手」から「自分のパソコン内の資料を一緒に見ながら作業してくれる相手」に変えられることです。

たとえば、営業メモ、日報、提案書、メールテンプレートが入ったフォルダをClaude Codeに開かせて、
「この内容をもとに営業メールの下書きを作って」と依頼できます。

するとClaude Codeは、必要なファイルを確認し、内容を整理し、メールの下書きという成果物まで作成します。

| 観点 | チャット型AI | Claude Code |
| --- | --- | --- |
| 使い方 | 画面上で質問や相談をする | 作業フォルダを開いて、実務を依頼する |
| 情報の渡し方 | 必要な文章やファイルを都度渡す | フォルダ内の資料、メモ、CSV、下書きなどをまとめて読ませる |
| 得意なこと | その場で回答を返す | ファイルを確認し、整理し、成果物として残す |
| 例 | 「この文章を要約して」 | 「このフォルダの営業メモを読んで、メール下書きを作って」 |

### 依頼のイメージ

たとえば、次のように依頼できます。

ここに出てくるSkill、メモリ、コネクタは、この手順書の後半で実際に一緒に設定しながら学んでいきます。最初は「こういう使い方ができる」という全体像だけつかめれば大丈夫です。

> Gmailを使って、株式会社ABCの佐藤様宛の営業メール下書きを作成してください。  
> このフォルダの営業ルール、メールテンプレート、日報、商談メモを確認してください。  
> メモリに入っている会社情報とメール署名も使ってください。  
> 営業メール作成用のSkillが使える場合は、そのルールも反映してください。  
> Gmailの下書きとして保存し、送信はしないでください。

この場合、Claude Codeは次の順番で進めます。

1. フォルダ内の営業ルール、テンプレート、日報、商談メモを確認する
2. メモリに入っている会社情報とメール署名を確認する
3. Skillのルールに沿って営業メールの文面を作る
4. Gmailコネクタを使って、下書きとして保存する
5. 送信はせず、ユーザーが確認できる状態にする

このように、単に回答を返すだけではなく、手元のファイル、メモリ、Skill、コネクタを組み合わせて、実際の業務に近い成果物を残せる点がClaude Codeデスクトップの特徴です。

### Chat、Cowork、Codeの違い

Claudeデスクトップには、主にChat、Cowork、Codeという使い方があります。

| 機能 | 使う場面 | 具体例 |
| --- | --- | --- |
| Chat | その場で相談したいとき | 文章の要約、メール文面の相談、アイデア出し |
| Cowork | Claudeに複数ステップの業務を任せたいとき | ローカル資料を読ませて営業メールを作る、調査結果を資料にまとめる、ファイルを整理する |
| Code | ソフトウェア開発やコードを扱う作業をしたいとき | プログラム修正、Webサイトのコード編集、エラー調査、開発用ファイルの確認 |

非エンジニアの方が、営業資料、日報、テンプレート、Gmailなどを使って業務を進める場合は、まずCoworkを使うと考えると分かりやすいです。

Codeは名前の通り、主にエンジニア向けの開発作業に使われます。ただし、Markdown、CSV、テキストファイルなどを扱うこともできるため、ファイル構造を学ぶ章ではCodeの考え方も一部参考になります。

迷ったときは、次の基準で選びます。

- 会話だけで済むなら、Chatを使う
- 業務資料や外部サービスを使って作業してほしいなら、Coworkを使う
- プログラムやWebサイトのコードを扱うなら、Codeを使う

本手順書では、非エンジニアの経営者や企業担当者でも実務で使えるように、Claude Codeデスクトップの基本操作、Skill、メモリ、コネクタ、具体的な活用例を順番に説明します。
