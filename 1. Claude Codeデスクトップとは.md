## 1. Claudeデスクトップとは

### まずはChatから使う

Claudeデスクトップには、Chat、Cowork、Codeなどの使い方があります。

この手順書では、まず多くの人が馴染みやすい `Chat` から始めます。

Chatは、Claudeと会話しながら文章作成、要約、相談、アイデア出しを行う使い方です。普段のChatGPTやClaudeのWeb版に近い感覚で使えるため、最初に操作に慣れる場所として向いています。

### Chatで学ぶこと

この手順書の「5. コネクタを使ってみよう.md」までは、Chatを使いながら次の機能を試します。

| 機能 | 何に使うか | この手順書で試すこと |
| --- | --- | --- |
| Skill | 何度も使う作業手順を覚えさせる | 営業メールの書き方を登録する |
| メモリ | よく使う情報を覚えさせる | 会社情報やメール署名を登録する |
| コネクタ | 外部サービスとつなぐ | Gmailの下書きを作る |

これらは、CoworkやCodeだけで使うものではありません。

Chatでも、Skill、メモリ、コネクタを使って、日常業務に近い作業を進められます。

### 依頼のイメージ

たとえば、3章で営業メール用のSkillを作り、4章で会社情報をメモリに入れ、5章でGmailコネクタを接続すると、Chatで次のように依頼できます。

```text
Gmailを使って、株式会社ABCの佐藤様宛の営業メール下書きを作成してください。

メモリに入っている会社情報とメール署名を使ってください。
営業メール作成用のSkillが使える場合は、そのルールも反映してください。
```

この場合、Claudeは次の順番で進めます。

1. メモリに入っている会社情報とメール署名を確認する
2. Skillのルールに沿って営業メールの文面を作る
3. Gmailコネクタを使って、下書きとして保存する
4. 送信はせず、ユーザーが確認できる状態にする

まずはこのように、Chat上で「相談する」「文章を作る」「外部サービスの下書きまで作る」流れを体験します。

### Chat、Cowork、Code、Cursorの違い

この手順書では、それぞれを次のように使い分けます。

| 使うもの | できること | 向いている場面 | 注意点 |
| --- | --- | --- | --- |
| Chat | Claudeと会話する。Skill、メモリ、コネクタも使える | 文章の要約、メール文面の相談、アイデア出しなどの単発タスク | メモリは使えるが、基本的には会話ごとのコンテキストで進む。長く続く案件管理やローカルフォルダ前提の作業には向きにくい |
| Cowork | ローカルのフォルダやファイルをプロジェクトに追加し、Claudeに複数ステップの作業を任せられる | 業務フォルダを読ませて相談内容を整理する、質問リストを作る、成果物を保存する | UIは非エンジニアにも分かりやすいが、人間が画面上でローカルファイルを手動編集するためのツールではない |
| Code | Coworkに近いAIエージェント機能に加えて、ファイル編集、差分確認、ターミナル、アプリのプレビューなど開発向け機能が使える | プログラム修正、Webサイトのコード編集、エラー調査 | 開発者向け。WindowsでClaude DesktopのCodeタブからローカルセッションを使う場合は、Git for Windowsのセットアップが必要になる |
| Cursor | 人間がローカルファイルやフォルダを直接開いて編集できる。必要に応じてAIエージェントにも作業を頼める | Markdownの手直し、ファイル作成、フォルダ整理、文章の修正 | 開発者向け機能も多い。初心者はまずMarkdownやテキストの手動編集に絞って使う |

非エンジニアの方は、次の順番で学ぶと分かりやすいです。

1. まずChatで、Claudeの基本操作、Skill、メモリ、コネクタに慣れる
2. 次にCoworkで、ローカルの業務フォルダを使った作業を任せる
3. 必要に応じてCursorで、作成されたファイルを自分の手で編集する

Codeは便利ですが、この手順書では主役にしません。
プログラム開発やWebサイト修正など、開発者向けの作業で使うものとして覚えておけば十分です。
