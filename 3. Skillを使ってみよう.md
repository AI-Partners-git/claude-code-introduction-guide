## 3. Skillを使ってみよう

### Skillとは

Skillとは、Claudeに追加できる「作業手順書」のようなものです。

たとえば、毎回Claudeに次のように伝えているとします。

> 営業メールは、売り込み感を強くしすぎず、相手が返信しやすい文面にしてください。  
> 件名は短くし、本文は5行以内にしてください。  
> 最後は「来週15分ほどお時間をいただけますでしょうか」のように、相手が次に取る行動を1つだけ書いてください。

このような毎回使う指示をSkillにしておくと、Claudeが必要な場面でそのルールを読み込み、同じ書き方で文章を作れるようになります。

| 例 | Skillに入れる内容 |
| --- | --- |
| 月次レポート作成 | 売上、粗利、案件数、問い合わせ数を前月比で見る。大きく増減した数字は理由を一言添え、最後に「今月やること」を3つに絞る |
| 稟議書・決裁資料レビュー | 目的、金額、期待効果、リスク、代替案が入っているか確認する。決裁者が最初に読む要約は3行以内にし、不足している情報を箇条書きで指摘する |
| 商談後レポート作成 | CRMや議事録ツールに接続し、会社名、担当者、課題、次回アクションを取得する。内容を要約し、`商談レポート_会社名_日付.md` というファイル名で保存する |


### 普通のプロンプトとの違い

普通のプロンプトは、その会話の中だけで使う一回限りの指示です。

Skillは、同じ作業を何度も行うための共通ルールです。毎回長い説明を書かなくても、Claudeが必要なときに読み込んで使います。

| 種類 | 向いている場面 | 例 |
| --- | --- | --- |
| 普通のプロンプト | その場だけの依頼 | 「この文章を短くして」 |
| Skill | 何度も使う作業ルール | 「当社の営業メールの型で作って」 |

以下のような場合にSkillを作成すると、定型業務の自動化や、Skillの共有ができます。

- 何度も同じ指示を出している場合
- 会社や部署で統一したい作業ルール

### データ学習に使われないように設定する

Skillには、会社の業務ルールや文体ルールを入力することがあります。
その前に、Claudeのモデル改善に会話データを使わせない設定を確認します。

1. Claudeデスクトップを開く
2. 左下のアカウント名を押す
3. `設定` を開く
4. `プライバシー` を開く
5. `AIモデルの改善にご協力ください`をオフにする

この設定をオフにすると、新しいチャットやコーディングセッションは、今後のモデル学習には使われません。

### カスタムSkillを作ってみる

Skillは、Claudeデスクトップの「カスタマイズ」から追加・管理できます。

ここでは、非エンジニアでも作りやすい例として「sales-email」というSkillを作ります。

1. Claudeデスクトップを開く
2. 「カスタマイズ」から「Skills」を開く
3. 「+」ボタンを押す
4. スキル名を入力する
    ```
    sales-email
    ```
5. スキルの説明を入力する
    - 何をするスキルか？いつ使うか？を記載します
    - スキル名を指定しない場合もAIがこの「説明」を参照し、登録されたスキルの中からどのスキルを使用するか判断します
    ```
    営業メールを、経営者向けに短く、売り込み感を抑えた文面で作成するSkillです。初回提案、商談後のフォロー、日程調整依頼のメールを作るときに使います。
    ```
6. スキルの手順を入力する
    ```
    営業メールを作成するときは、次のルールに従ってください。

    - 宛名は「会社名 + 名前 + 様」で始める
    - 件名は20文字前後にする
    - 本文は5行以内にする
    - 1文は60文字以内を目安にする
    - 専門用語はなるべく使わない
    - 売り込み感の強い表現は避ける
    - 相手が返信しやすい、軽い確認のトーンにする
    - 最後に、日程調整または資料確認の依頼を1つだけ入れる
    - 署名は会社指定のものを使う
    ```
7. 保存する

スキル名は、日本語ではなく半角英数字、ハイフン、アンダースコアで記載する必要があります。日本語の説明は、説明欄や手順欄に書きます。

保存できたら、Skills画面の一覧に作成したSkillが表示されます。使わないSkillは、同じ画面でオフにできます。

### 作ったSkillを試す

Skillを追加したら、Chatを開いて次のように依頼します。

```text
sales-email Skillを使って、営業メールを作ってください。

相手：
株式会社ABC
佐藤様

目的：
先日紹介した業務効率化サービスについて、来週30分ほど説明の時間をいただきたい。

補足：
相手は中小企業の経営者です。
強い売り込みではなく、軽い確認のトーンにしてください。
```

うまく動いていれば、ChatでもSkillで指定したルールに沿ったメールが作られます。

補足：「sales-email Skill」とSkill名を明示しなくても営業メールを作って伝えるだけで、AIが依頼に合うSkillを自動で選んで使用してくれます。
ただし、最初に試すときや確実に使わせたいときは、「sales-email のルールを使ってください」と書くと確実です。

次の点を確認します。

- 件名が短いか
- 本文が5行以内か
- 売り込み感が強すぎないか
- 最後の依頼が1つだけになっているか

期待する出力と違う場合は、Skillの手順を修正します。たとえばメールが長すぎる場合は、「本文は5行以内にする」「1つのメールで伝える要件は1つに絞る」といったルールを追加します。

### 作ったSkillが一覧に表示されない場合

Claudeに普通に「Skillを作って」と依頼すると、`skill.md` というファイルだけが作成されることがあります。この状態では、Claudeデスクトップのカスタマイズにはまだ登録されていません。

カスタマイズの一覧に表示するには、次の対応が必要です。

- Claudeデスクトップの「カスタマイズ > Skills > +」から、スキル名、説明、手順を入力して保存する

非エンジニアの場合は、最初からカスタマイズの画面で作成する方法がわかりやすいです。

### Skillを作るときの注意点

Skillには、会社の業務ルールや文体ルールを入れると便利です。一方で、何でも入れればよいわけではありません。

- APIキー、パスワード、個人情報は入れない
- 顧客情報をそのまま入れない
- 1つのSkillに複数の業務を詰め込みすぎない
- まずは文章作成や資料チェックなど、影響範囲が小さい業務から試す
- 会社で共通利用する場合は、管理者または責任者が内容を確認する

特に、外部からダウンロードしたSkillを使う場合は注意が必要です。信頼できる提供元か確認し、中身を読んでから有効にしてください。
